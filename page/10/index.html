你已经知道如何用组合键停止 shell 中正在运行的作业，在作业停止后，Linux 系统可以让你无条件终止或重启它。你可以用 kill 命令终止该进程，要重启停止的进程需要想其发送一个 SIGCONT 信号。

Linux 控制脚本

目前为止，我们运行脚本的唯一方式就是以实时模式在命令行界面上直接运行。但这并不是 Linux 上运行脚本的唯一方式，还有其它一些运行 shell 脚本的选择。本章我们会介绍控制何时以及如何在系统上运行 shell 脚本的不同方法。

处理信号

Linux 通过信号来在运行在系统上的进程之间通信。默认情况下，bash shell 会忽略收到的任何 SIGQUIT 和 SIGTERM 信号。但是 bash shell 会处理收到的 SIGHUP 和 SIGINT 信号。bash shell 允许用键盘上的键组合生成两种基本的 Linux 信号。这个特性在需要停止或暂停跑飞了的程序时就能派上用场。

  • 终止进程 Ctrl+C 组合键会生成 SIGINT 信号，会停止 shell 中运行的当前进程。
  • 暂停进程 Ctrl+Z 组合键会生成 SIGTSTP 信号，会暂停一个进程，跟终止不同，暂停进程会让程序继续保留在内存中，并能从上次停止的位置继续运行。

你也可以不忽略信号，在信号出现时捕捉它们并执行其它命令。trap 命令允许你来指定 shell 脚本要观察哪些 Linux 信号并从 shell 中拦截。如果脚本收到了 trap 命令中列出的信号，它会阻止它被 shell 处理，而在本地处理它。

以后台模式运行脚本

直接在命令界面运行脚本有时不怎么方便，好在我们有一个很简单的方法，即在命令后加 & 符号，在后台模式运行 shell 脚本。在后台模式中，进程运行时不会和终端会话上的 STDIN、STDOUT 以及 STDERR 关联。

当 & 符放到命令后时，它会将命令和 bash shell 分离开来，并将它作为系统上的独立后台进程运行。每次启动新作业时，Linux 系统都会为其分配一个新的作业号和 PID，在终端会话中使用后台进程时，必须特别小心。每个后台进程都绑定到了该终端会话上，如果进程会话退出了，后台进程也会退出。如果你有关联到该终端的还在运行的后台进程，有的终端模拟器会提醒器，有的甚至不会，如果你想退出控制台后，脚本继续以后台形式运行，我们有其它办法解决，看下节。

在非控制台下运行脚本

有时需要在终端会话中启动的脚本一直在后台运行，直到其完成，即使你退出了终端会话。可以用 nohup 命令来实现。

nohup 命令运行另外一个命令来阻断所有发送给该进程的 SIGHUP 信号。这会在退出终端会话时阻止进程退出。由于 nohup 命令会从终端解除进程的关联，进程会丢掉到 STDOUT 和 STDERR 的链接，为了保存该命令产生的输出，nohup 命令会自动将 STDOUT 和 STDERR 的消息重定向到另一个名为 nohup.out 的文件中。在运行同目录中的多个命令时，因为所有的输出都会发送到同一个 nohup.out 文件，可能会有点叫人困惑。

Linux 结构化命令

默认情况下 shell 按照出现的次序来处理 shell 脚本中的每个单独命令。对于顺序操作来说已经足够了，然后许多程序要求在 shell 脚本中的命令间有一些逻辑流控制。有一类命令会基于变量值或者其它变量的结果等条件使脚本跳过或循环执行命令。这样的命令常常称为结构化命令。
结构化命令允许你改变程序执行的顺序，在某些条件下执行一些命令而在其它条件下跳过另一些命令。

Linux 文件权限

1. Linux 的安全性

Linux 安全系统的核心是用户帐号。每个能进去 Linux 系统的用户都会被分配一个唯一的用户帐号。用户对系统上对象的访问权限取决于他们登入系统时用的帐号。
用户权限是通过创建用户时分配的用户ID（UID）来跟踪的。UID是个数值，每个用户都有个唯一的UID，但在登录系统时不使用UID而是登录名。
Linux 系统使用特定的文件和工具来跟踪和管理系统上的用户帐号。

1.1 /etc/passwd 文件

Linux 系统使用一个专门的文件来将用户的登录名匹配到对应的UID值。这个文件就是 /etc/passwd 文件。/etc/passwd 文件的字段包含了如下信息：登录用户名、用户密码、UID、GID、备注字段、用户HOME目录位置、用户默认Shell。

1.2 添加新用户

添加新用户到 Linux 系统的工具是 useradd。这个命令提供了一次性创建新用户帐户及设置用户 HOME 目录结构的简便方法。useradd 命令使用系统的默认值以及命令行参数来设置用户帐户。可以用 useradd 的命令加 -D 参数来查看你的 linux 系统的系统默认值：

-D 参数显示了在创建新用户时如果你不在命令行指定的话 useradd 命令将使用的默认值。

1.3 删除用户

如果想从系统中删除用户，userdel 可以满足这个需求。默认情况下，userdel 命令会只删除 /etc/passwd 文件中的用户信息，而加上 -r 参数，userdel 会删除用户的 HOME 目录以及 mail 目录。

1.4 修改用户

  • usermod

usermod 命令是用户帐户修改工具中最为强大的一个，它能用来修改 /etc/passwd 文件中的大部分字段，只需用与想修改的字段对应的命令行参数就可以了。参数大部分跟 useradd 命令的参数一样，比如 -c 用来修改备注字段，-e 用来修改过期日期，-g 用来修改默认的登录组，-l 用来修改用户帐户的登录名，-L 用来锁定帐户，-p 用来修改帐户的密码，-U 用来解除帐户锁定。

  • passwd 和 chpasswd

改变用户密码的一个简便方法就是用 passwd 命令。如果只用 passwd 命令，它会改变你自己的密码。系统上的任何用户都能改变它自己的密码，但只有 root 用户才有权限改变别人的密码。-e 选项能强制用户下次登录时修改密码。
如果需要为系统中的大量用户来修改密码，chpasswd 命令能让事情简单许多。chpasswd 命令能从标准输入自动读取登录名和密码对列表，给密码加密，然后为用户帐户设置。

  • chsh、chfn 和 chage

chsh 用来快速修改默认的用户登录 shell，使用时必须用 shell 的全路径名作为参数，不能只用 shell 名。
chfn 命令提供了在 /etc/passwd 文件的备注字段中存储信息的标准方法。
chage 命令用来帮助管理用户帐户的有效期。chage 命令中好用的功能是设置帐户过期日期。通过它，你就能创建临时用户了。

2. 使用 Linux 组

用户帐户在控制单个用户安全性方面很管用，组权限用户则允许多个用户共享一组公用的权限来访问系统上的对象，比如文件、目录或设备等。
每个组都有一个唯一的GID，在系统上这是一个唯一的数值。

2.1 /etc/group 文件

类似用户帐户信息，/etc/group 文件包含系统上用到的每个组的信息。

  • 创建新组

groupadd 命令用来在系统上穿件新组。创建新组时，默认没有用户属于该组。可以用 usermod 命令来添加用户到组。

  • 修改组

groupmod 命令可以修改已有组的GID或组名，修改组名时，GID和组成员不会变，只有组名会改变。由于所有的安全权限都是基于GID的，可以随意改变组名而不影响文件的安全性。

3. 理解文件权限

3.1 使用文件权限符

可以用 ls 命令用来查看 Linux 系统上的文件、目录和设备的权限。

3.2 默认文件权限

umask 命令用来设置用户创建文件和目录的默认权限。touch 命令用分配给我的用户帐户的默认权限创建了这个文件。

Linux 定位系统环境变量

Linux 系统用环境变量来在程序和脚本中标识它自己。这为你的程序提供了获得信息的一个简便方法。问题是如何设置这些变量。
在你登录 Linux 系统启动一个 bash shell 时，默认情况下 bash 在几个文件中查找命令。这些文件称作启动文件。bash 检查的启动文件取决与你启动 bash shell 的方式。启动 bash shell 有3种方式：

  • 登录时当作默认登录 shell；
  • 作为非登录 shell 的交互式 shell；
  • 作为运行脚本的非交互式 shell。

Linux GNU 工具链

操作系统除了有内核来控制硬件设备外，还需要工具链来执行一些标准功能，比如控制文件和程序。
GNU组织开发了一套完整的Unix工具链，但没有可以运行它们的内核系统。这些工具链是在开源软件的软件开发理念下开发的。
开源软件理念允许程序员开发软件并将其免费发布。任何人都可以使用、修改该软件，或将该软件集成进自己的系统，而无需支付任何授权费用。将linux内核系统和GNU工具整合起来，就可以创造一个完整的、功能丰富的免费操作系统。
尽管通常我们将linux内核和GNU工具链的结合体称为Linux，但也有一部分linux纯粹主义者将其称为GNU/Linux系统来表彰GNU组织为此所作出的贡献。
