Rsync 同步常用命令

-t 选项

  1. 使用 -t 选项后，rsync 总会想做一件事，就是将源文件的 “modify time” 同步到目标机器
  2. 带有 -t 选项的 rsync 会变得更聪明些，它会在同步前先对比两边文件的时间戳和大小，如果一致，则认为两边文件一样，对此文件就不采取更新动作

-v 选项

让 rsync 输出更多的信息，越多的 v 可以获得越多的日志信息

-z 选项

使用这个选项，rsync 就会把发向对端的数据先进行压缩再传输

-r 选项

使用这个选项，rsync 会同步文件夹

-p 选项

保持权限的意思。如果不使用此选项，rsync 这样处理权限问题：

  1. 如果目的端没有此文件，那么在同步后将目的端文件权限保持与源端一致
  2. 如果目的端存在此文件，那么只同步文件内容，权限报纸不变

-o -g 选项

这两个选项是一对，用来保持文件的属组和属主

–progress 选项

使用这个选项，rsync 会显示出传输进度信息

–delete、–delete-excluded、–delete-after

  1. delete 如果源端没有此文件，则删除目的端的此文件，需要搭配 -r 使用
  2. delete-excluded 指定一些要在目的端删除的文件
  3. delete-after 默认情况下，rsync 会先清理目的端的文件再开始数据同步，如果使用此选项，则 rsync 会先进行数据同步，都完成后再删除那些需要清理的文件

–partial 选项

断点续传，默认情况下，rsync 会删除那些传输中断的文件，然后重新传输，但在一些特别情况下，我们不希望重传而是续传

Find atime ctime mtime 用法与区别总结

atime、ctime和mtime

atime 是指 access time，即文件被读取或者执行的时间，修改文件是不会改变 access time 的。

ctime 即 change time 文件状态改变时间，指文件的节点被修改的时间。

mtime 即 modify time，指文件内容被修改的时间。

find -mtime 中的参数 n

find . -mtime n 中的n指的是 24n，+n、-n、n分别表示大于、小于等于24n。

我们看看默认的配置文件是怎样的：

  1. normal_check_interval 表示在正常情况下每10分钟检查一次服务状态

Nagios 主机监控与报警时间间隔

我们看看默认的配置文件是怎样的：

  1. 在 Ok 状态下，Nagios 用 check_interval 定义的时间间隔来监控；
  2. 出现问题后，切换为 retry_interval 和 max_check_attempts 进行监控，达到 max_check_attempts 后触发首次报警；
  3. 同时恢复为 check_interval 进行监控，并使用 notification_interval 定义的时间间隔来发送报警；
  4. 服务恢复后，在最近的 check_interval 点发送报警信息，完成报警周期。

注意：

  1. max_check_attempts 定义为 1，检测到问题立刻报警，不重试；
  2. notification_interval 定义为 0，报警只发送一次，不重发。

Shell 中中括号的用法

单中括号

  1. bash 的内部命令，[ 和 test 是等同的。如果我们不用绝对路径指明，通常我们用的都是 bash 自带的命令。if/test 结构中的左中括号都是调用 test 的命令标识，右中括号是关闭条件判断的。这个命令把它的参数作为比较表达式或者作为文件测试，并且根据比较的结果来返回一个退出状态码。

  2. Test 和 [] 中可用的比较运算符只有 == 和 !=，两者都是用于比较字符串的，不可用于整数比较，整数比较只能使用 -eq、-gt这种形式。无论是字符串比较还是整数比较都不支持大于号小于号。如果实在想用，对于字符串比较可以使用转义形式。[] 中的逻辑与和逻辑或使用 -a 和 -o 表示。

双中括号

  1. bash 的关键字，并不是一个命令。[[]]结构比[]更加通用。在[[和]]之间所有的字符都不会发生文件名扩展或者单词分隔，但是会发生参数扩展和命令替换。

  2. 支持字符串的模式匹配，使用 =~ 操作符甚至支持 shell 的正则表达式。字符串比较时可以把右边作为一个模式，而不仅仅是一个字符串，比如 [[ hello == hell? ]]，结果为真。[[]]中匹配字符串或通配符，不需要引号。

  3. 使用[[…]]条件判断结构，而不是[…]，能够防止脚本中的许多逻辑错误。比如 &&、||、< 和 > 操作符能够正常存在于 [[]]条件判断结构中。

  4. bash 把双括号中的表达式看作一个单独的元素，并返回一个退出状态码。

Shell 中花括号的用法

${var}

当变量名和后面的内容都是变量命名所允许的内容时，这个时候直接使用$var是不行的，得用{}把变量名括起来。

${var:-string}

当变量var值为空时${var:-a}会把a作为值返回，当变量不为空时，${var:-a}会把var的值作为结果返回
注意：表达式并会不修改变量var的值

${var:+string}

${var:+a}的替换规则跟上面相反，即当变量var的值不为空的时候将值替换成a，当变量var为空时则不替换

${var:=string}

当变量var的值为空时，${var:=a}会将其赋值为a，且变量var也被赋值是a了，则${var:=a}的值就是变量var的值

${var:?string}

若变量var不为空，则用变量var的值来替换 ${var:?string}；若变量var为空，则把string输出到标准错误中，并从脚本中退出，我们可利用此特性来检查是否设置了变量的值。

${var%pattern}、${var%%pattern}

${variable%pattern}，shell在variable中查找，看是否有给定的pattern模式结尾，如果是，就从命令行把variable中的内容去掉右边最短的匹配模式
${variable%%pattern}，shell在variable中查找，看是否有给定的pattern模式结尾，如果是，就从命令行把variable中的内容去掉右边最长的匹配模式

${var#pattern}、${var##pattern}

${variable#pattern}，shell在variable中查找，看是否有给定的pattern模式开始，如果是，就从命令行把variable中的内容去掉左边最短的匹配模式
${variable#pattern}，shell在variable中查找，看是否有给定的pattern模式开始，如果是，就从命令行把variable中的内容去掉左边最长的匹配模式

扩展

a的位置可以使用变量名或者一个命令，并不一定只能是常量。

{1,2,3}和{1..n}
