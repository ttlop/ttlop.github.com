Zabbix Web Monitoring

Overview

With Zabbix you can check several availability aspects of web sites.

To perform web monitoring Zabbix server must be initially configured with cURL(libcurl) support.

To activate web monitoring you need to define web scenarios. A web scenario consists of one or several HTTP requests or “steps”. The steps are periodically executed by Zabbix server in a pre-defined order.

Since Zabbix 2.2 web scenarios are attached to hosts/templates in the same way as items, tirggers, etc. That means that web scenarios can also be created on a template level and then applied to multiple hosts in one move.

Zabbix 的 system.cpu.load 与 CPU 使用率

CPU 使用率

CPU 使用率显示的是程序在运行期间实时占用的 CPU 百分比

CPU load

CPU load 显示的是一段时间内正在使用和等待使用 CPU 的平均任务数，即负载。所以 CPU 使用率高，并不意味着负载就一定大。

举例说明

  1. 公司上下班需打卡，每次只能有一个人在打卡且限定 5 秒钟，其他人等待，每人限定打一次，若一次不成功，只能重新排队，等待下一轮打卡

  2. 打卡机就相当于 CPU，而正在或等待打卡的人就相当于任务数

  3. 在打卡过程中，肯定会有人打完就走掉，有人没有打完而选择重新排队打卡，更会有新增的人在这儿排队，这个人数的变化就相当于任务数的增减

  4. 为了统计平均负载情况，我们每 5 秒统计一次人数，并在第1、5、15分钟的时候对统计情况取平均值，从而形成第1、5、15的平均负载

  1. 有的人拿起打卡机就打卡，一直到 5 秒钟结束，而有的人可能前 2 秒没有操作，后 3 秒才真正在打卡

  2. 如果把打卡机比作CPU， 任务看作任务，我们就说前一个人的 CPU 利用率高，后一个人的 CPU 利用率低，但无论 CPU 的利用率高低，跟后面有多少任务在排队没有必然关系

  3. 除非后一秒打卡的人因为浪费了时间导致打卡不成功需要重新排队打卡，由于排队人数相当于任务数，任务数增加会导致负载变大，此时 CPU 的利用率低才会对负载有影响
