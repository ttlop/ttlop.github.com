Zabbix Triggers 触发器

触发器使用逻辑表达式来评估通过 item 获取得到数据是处于哪种状态。Item 每次获取到一个新值都会使用触发器表达式去计算它的状态，在触发器表达式中我们可以定义哪些值是合理的，哪些是不合理的，如果出现不合理的值，触发器会把状态改为 PROBLEM。

1. 创建触发器

  1. 点击 Configuration -> Hosts

  2. 点击 Hosts 相关行的 Triggers

  3. 点击右上角的 Create trigger，你也可以修改列表中的触发器

  4. 在表单中输入相应的信息

2. Trigger 表单

参数 描述
Name 触发器名称，名称可以包含宏变量，$1,$2..$9可以用来关联表达式的常量
Expression 计算触发器状态的逻辑表达式
Multiple PROBLEM enents generation 通过设置该选项，你可以在触发器产生 problem 的时候触发一个事件
Description 触发器的描述，一般 name 写的不清楚，这边可以具体描述这个触发器的作用
URL 在 Monitoring->Triggers 中，可以看到 URL 并且可以点击，一般情况下它需要配置触发器ID和主机ID来使用，例如：triggers.php?form=update&hostid={HOST.ID}&triggerid={TRIGGER.ID}，这个地址是我从 Trigger 表单的 URL 栏获取到的，其中使用 {HOST.ID} 和 {TRIGGER.ID} 宏变量来替换主机ID和触发器ID，这样可以直接点击到具体触发器中
Severity 设置严重性级别，后面我们根据不同的严重级别来定义不同的触发事件
Enabled 当前触发器是否启用

2.1 Tigger severity

Severity 通常用来定义当前 item 的一个状态严重性。我们根据不同的严重性来定义不同的时间，例如报警。

Zabbix 自带如下严重性定义。

严重性 定义 颜色
Not classified 未知 灰色
Information 一般信息 浅绿
Warning 警告 黄色
Average 一般问题 橙色
High 严重问题 红色
Disaster 灾难，会带来损失那种 深红

2.2 Severity 用途

*. 可视化显示，不同级别显示不同颜色

*. 声音报警，不同级别不同声音

*. 使用用户自定义媒体报警，例如严重问题发短信，其它问题发送邮件

*. 根据严重性来定义是否报警

3. Zabbix 触发器依赖关系

触发器可以设置依赖性，例如配置了两个触发器，一个触发器监控主机是否运行，一个触发器监控网络是否通畅。假如网络出现故障，主机并未出现故障，我们依旧会收到这两个触发器给到的故障通知。

现在的问题很明显，主机是正常的，肯定不希望收到它的故障信息。所以在配置触发器的时候，我们需要增加依赖关系，依赖网络是否畅通这个触发器，一旦网络出现故障，将不会触发主机故障的通知。单个触发器可以依赖于多个触发器。

3.1 依赖要点

*. 一个主机的触发器可以依赖其它主机的触发器，但是不要有死循环

*. 一个模板的触发器可以依赖另外一个模板的触发器

*. 模板中的触发器可以依赖主机中的触发器，主机中的触发器不能依赖模板中的触发器
