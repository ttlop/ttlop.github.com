MikroTik 基于 IPv6 的 6to4 通道

6to4 采用特殊的 IPv6 地址使在 IPv4 海洋中的 IPv6 孤岛能相互连接。此时 IPv6 的出口路由器与其它的 IPv6 域建立隧道连接。IPv4 隧道的末端可从 IPv6 域的地址前缀中自动提取，通过这个机制，站点能够配置 IPv6 而不需要向注册结构申请 IPv6 地址空间。（也就是说，即使没有 IPv6 地址，通过 6to4 通道，别人也可以通过你申请的 IPv6 的访问，但是这个 IPv6 地址为动态）

第一步

从 https://www.tunnelbroker.net/ 网站申请一个帐号，步骤略，会将帐号密码发送到填写的邮箱。

第二步

选择 Create Regular Tunnel 选项，创建通道

  1. 首先我们在 IPv4 Endpoint 右边的框内填写你的外网IP地址！

  2. 选择一个可用的通道与你建立连接，我选择的是 Tokyo, JP。这里你可以根据网速情况选择速度稍微好一点的通道服务。

  3. 选好后，点 Create Tunnel 开始创建通道！根据个人网速，这个需要等待的时间不一样。

  4. 创建好通道后，我们点击 Example Configurations，就可以看到我们熟悉的 ROS 代码命令了！

第三步

打开我们熟悉的 WinBox，点击 New Terminal，然后将上述代码复制到里面去，注意检查命令是否有错误并且查看执行情况！

  1. 上述三行代码都添加上去后，打开网页！还是不能访问 IPv6 站点，为什么呢？这是因为没有做路由！路由器只知道你要用 IPv6 地址来访问 HE 提供的通道，但是它却不知道你要用 IPv6 通过 HE 的 6to4 路由器实现对其它网站的访问！

  2. 修改DNS，建议修改辅助 DNS 为谷歌的，主 DNS 还是设置为网络服务商的。这样访问国内站点也没有影响。

  3. 添加一条对外的路由，下述编号为 0 的即为我们添加的 IPv6 缺省路由。

注

答：此处需要检查你的路由器内网（LAN）是否设置了指定了路由前缀 Routed IPv6 Prefixes
