Linux 获取当前正在执行脚本的绝对路径

常见的误区是使用 pwd 命令，该命令的作用是 pwd - print name of current/working directory，这才是此命令的含义，当前的工作目录。这里没有任何意思说明，这个目录就是脚本存放的目录，我们可以如下测试：

你会发现，显示的是执行命令的路径 /root，而不是脚本所在的路径 /tmp。

另一个错误的答案是 $0，这个也是不对的，这个 $0 是 Bash 环境下的特殊变量，真实含义如下：

  1. 使用一个文件调用 bash，那么 $0 的值是那个文件的名字，不是绝对路径咯
  2. 使用 -c 选项启动 bash 的话，真正执行的命令会从一个字符串中读取，字符串后面还有别的参数的话，使用从 $0 开始的特殊变量引用
  3. 除此之外，$0 会被设置成调用 bash 的那个文件的名字，没说是绝对路径咯

下面是正确答案

在此进行解释如下：

  1. dirname $0，取得当前执行的脚本文件的父目录
  2. cd dirname $0，进入这个目录（切换到当前工作目录）
  3. pwd，显示当前工作目录（cd执行后的结果）

由此，我们获得了当前正在执行的脚本的存放路径。
